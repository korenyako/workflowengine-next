## Introduction

We’ve tested the performance of Workflow Engine and are happy to share the results with you. The tests were conducted on Windows and Linux. The standard version without further optimizations was tested.

Here we go…

## Test description

We’ve measured the performance with SoapUI, a free automated testing tool, using the Simple strategy ([SoapUI strategies](https://www.soapui.org/load-testing/simulating-different-types-of-load.html)) with different amount of threads: 1, 4, 8, and 16 threads. The control performance measurement was done over a set of 10 000 processes.

A scheme comprising three Activities and two Transitions is used in the test:

![Workflow Scheme](/images/blog/workflow-performance/workflow-scheme.png)

A single test consists of 14 steps executed successively. Each step in the test is a call of one of the basic API methods. You can read full description for Workflow Engine [here](https://workflowengine.io/documentation/main-terms/basic-operations/), and for Workflow Server [here](https://workflowengine.io/documentation/workflow-server/workflow-api/).

## Windows Test

We used a dedicated server with the following configuration for the test:

-   [Intel® Xeon® E3-1275 v5 Quad-Core Skylake (4 core, 8 threads)](https://ark.intel.com/products/88177/Intel-Xeon-Processor-E3-1275-v5-8M-Cache-3-60-GHz-)
-   HARD DRIVE 2 x 480 GB SSD SATA-III Data Center Series
-   RAM 64 GB DDR4 ECC
-   MS Windows 2016
-   MS SQL Server 2017 Developer Edition
-   .NET Core 2.1
-   Workflow Server 2.3

Workflow Server, the database and the testing tool were deployed to a single server in this test. 500 000 processes were created in the database before running the control test.

## Conclusion

Workflow Engine demonstrates perfect performance both on weak machines and powerful servers. A well-thought-through architecture ensures stable processing times even with default database settings and hundred thousands to millions of processes, including active/unfinished ones.

You can process up to 1000 basic operations per second with an average time of processing one request of 5 to 30 milliseconds on an average server. This is enough for any solution from corporate systems to IoT platforms.
