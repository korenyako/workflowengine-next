# Design rules

Свод визуальных правил сайта. Цель — стильный, минималистичный UI, построенный на типографике и смелых, но сдержанных решениях. Правила нормативные: новые блоки и компоненты им следуют, существующие постепенно приводятся в соответствие.

## Главный инвариант

**Граница элемента создаётся ровно одним способом:** либо контуром, либо контрастом фонов — не обоими сразу.

Из этого выводятся все правила ниже.

## 0. Типографика

### Семьи

- **Заголовки — DM Sans** (9pt opsz), `font-weight: 800`, `letter-spacing: -0.02em`, `line-height: 1.1`. Применяется ко всем h1–h6 по умолчанию (правило в [globals.css](../src/styles/globals.css)) и через класс `.font-heading`. Вес 800 — инвариант: **не использовать `font-bold`/`font-extrabold`/`font-semibold` поверх `font-heading`**; если нужна визуальная иерархия — меняется `font-size`, не `font-weight`.
- **Body — Inter**, `font-weight: 400`. Применяется к `body` и ко всем элементам без специального класса.
- **Code** — JetBrains Mono через `.font-mono`, `code`, `pre`.

Подключение — `next/font/google` в [layout.tsx](../src/app/layout.tsx), CSS-переменные (`--font-dm-sans`, `--font-inter`, `--font-jetbrains-mono`) маппятся в tokens Tailwind (`fontFamily.heading`, `fontFamily.sans`, `fontFamily.mono`) в [tailwind.config.ts](../tailwind.config.ts). Для заголовков референс — pencil.dev.

### Type scale

Фиксированный набор ролей. Каждый кусок текста на странице попадает ровно в одну роль — размеры и цвета выбираются **только** из этой таблицы.

| Роль | Классы | Где |
|---|---|---|
| **Display / h1** | `text-5xl sm:text-6xl lg:text-7xl xl:text-8xl font-heading text-slate-900` | Hero only |
| **Section h2** | `text-4xl lg:text-5xl xl:text-6xl font-heading text-slate-900` | Заголовок секции |
| **Card / h3** | `text-2xl lg:text-3xl font-heading text-slate-900` | Заголовок карточки, внутренний подзаголовок |
| **Eyebrow** | `text-sm uppercase tracking-[0.2em] font-mono font-medium text-[#4286F4]` | Метка-подпись над h2 (mono-шрифт, терминал-стиль) |
| **Lead** | `text-xl text-slate-600` | Описание под h1/h2 |
| **Body** | `text-lg text-slate-600` | Текст карточки, параграф |
| **Meta** | `text-sm text-slate-500` | Note, подпись, footer-мелочь |

**Запреты поверх `.font-heading`:**
- `font-bold`/`font-extrabold`/`font-semibold`/`font-medium` — вес 800 уже зашит в класс.
- `tracking-wide`/`tracking-wider`/`tracking-tight` — letter-spacing -0.02em уже зашит.
- `leading-tight`/`leading-snug`/`leading-normal` — line-height 1.1 уже зашит.
- `uppercase` — это атрибут eyebrow-роли, не heading.

**Цветовая палитра для текста:** только семейство `slate-*` + `blue-600` для акцентов. Ветвление `text-gray-* : text-slate-*` по `isLightBg` — legacy от тёмной темы FormEngine, удаляется (сайт только светлый).

## 1. Никаких теней

Ни на карточках, ни на блоках, ни на кнопках, ни на модалках, ни на дропдаунах. Элевация передаётся контрастом фонов и типографикой, не тенями.

Tailwind: ни `shadow-*`, ни `drop-shadow-*`, ни кастомных `box-shadow` в стилях. Единственное исключение — focus-ring (см. §5).

## 2. Hover — только через прозрачность

Разрешено: изменение `opacity`, затемнение (`hover:bg-*-darker`), высветление (`hover:bg-*-lighter`).

Запрещено: `hover:scale-*`, `hover:translate-*`, `hover:shadow-*`, появление/исчезновение элементов, смена иконок, анимации размеров.

То же правило — для `active` и `disabled`: только прозрачность/затемнение, никаких трансформаций.

## 3. Контуры — по фону

- **Есть фон** (карточка/блок/секция с `bg-*`) → **нельзя контур**. Граница уже создана контрастом фонов.
- **Нет фона** (прозрачный контейнер на общем фоне страницы) → **можно контур**, если элемент иначе растворяется.

Инпуты — частный случай: контур появляется, только если контраст фона инпута и окружения недостаточен. При большом контрасте фонов контур не нужен.

Медиа (картинки, видео, иллюстрации) — контур разрешён.

Hairline-разделители между секциями (`border-t`, `border-b` на уровне секций) — не считаются «контуром» и разрешены. Трактуем их как типографический элемент.

**Толщина контура — всегда 2px** (`border-2`). Tailwind default `border` (1px) и `border-4`+ не использовать. Это касается и кнопок (secondary), и карточек, и инпутов, и бейджей, и таблиц — единая толщина по всему сайту.

## 4. Кнопки

Все кнопки плоские, без теней, без градиентов, **полностью скруглённые** (`rounded-full`).

- **Primary** — с фоном (`bg-[#4286F4]`, `text-white`). Hover — затемнение фона (`hover:bg-[#2e6ad4]`).
- **Secondary** — контурная (`border-2 border-[#4286F4]`, `bg-transparent`, `text-[#4286F4]`). Hover — заливка `#4286F4` с `text-white`.

Форма pill-shape (`rounded-full`) применяется ко всем кнопочным CTA — включая ссылки-кнопки, CTA в hero, формах, карточках. Квадратные/скруглённые по мелкому радиусу кнопки — запрещены.

Canonical-имплементация — [Button.tsx](../src/components/Button.tsx).

## 5. Скругление углов у секций и блоков

Все секции, блоки, карточки — со скруглёнными углами. Острые углы запрещены.

Точные пропорции (малый/средний/крупный радиус, соотношение с размером блока) **будут определены позже**, после того как появится достаточно визуальных кейсов. До этого — использовать единый умеренный радиус на уровне `rounded-2xl` / `rounded-3xl`, последовательно в рамках одного экрана.

Исключение — кнопки: они `rounded-full` (см. §4), это отдельная типологическая форма.

Медиа (картинки, видео) — также со скруглением.

## 6. Исключение: focus-ring

Для клавиатурной навигации (`:focus-visible`) контур/кольцо рисуется **всегда**, включая залитые кнопки и элементы с фоном. Это единственный случай, когда правило §3 нарушается.

Причина — a11y: прозрачности недостаточно, чтобы показать фокус пользователю с клавиатуры.

## 7. Слайд-карточки (surface: card)

Контентные блоки страницы выстраиваются как крупные rounded-карточки на белом фоне страницы. Ритм — чередование card-блоков с плоскими flat-блоками (hero, логотипы, фичи-strip).

**Опт-ин через JSON:** блок в [main.json](../src/data/main.json) / `features.json` / `server.json` получает `"surface": "card"` в своих `props`. Обёртка ([blocks.tsx](../src/components/blocks.tsx) / [PageBlocks.tsx](../src/components/PageBlocks.tsx)) видит флаг и оборачивает компонент в:

```tsx
<div className="mx-4 sm:mx-6 lg:mx-8 my-6 lg:my-8 bg-slate-100 rounded-[40px] lg:rounded-[48px] overflow-hidden">
```

- **Боковые отступы:** `mx-4 sm:mx-6 lg:mx-8` (16→24→32px) — карточка не прилипает к краям.
- **Вертикальный ритм:** `my-6 lg:my-8` (24→32px) между соседними карточками.
- **Радиус:** `rounded-[40px] lg:rounded-[48px]` — крупный, а не `rounded-3xl` (24px). Это главная визуальная подпись слайдов.
- **Фон слайда:** `bg-slate-100` (#F1F5F9). Точный оттенок может эволюционировать в один токен (TBD — может быть warm beige типа `#F5F3EE`).

**Card-in-card:** если внутри card-блока используются внутренние карточки (`DetailedFeatureGridBlock`, `ProductsGridBlock`), внутренние карточки получают `bg-white` — чтобы контраст с родительским `bg-slate-100` сохранялся (правило §3 — граница одним способом, через контраст фонов).

**Hero / LogosBlock / FeaturesGrid остаются flat** — hero всегда full-bleed, логотипы узкая лента, `FeaturesGrid` делает визуальный «вдох» между card-ами. Block-компоненты не знают, в каком режиме их рендерят — их внутренний `<section className="py-16 px-4 sm:px-8">` работает и внутри card-обёртки (становится inner padding), и в flat-режиме.

**Футер — частный случай слайд-карточки.** Делит визуальный язык слайдов: `bg-slate-100`, боковые поля `mx-4 sm:mx-6 lg:mx-8` и верхний ритм `mt-6 lg:mt-8` такие же, как у card-блоков. Отличие — только верхние углы скруглены (`rounded-t-[40px] lg:rounded-t-[48px]`), нижние нет, и снизу нет `mb-*`: footer прилипает к низу viewport, по бокам сохраняя полосы страничного фона. Реализация — в [FooterBlock.tsx](../src/components/FooterBlock.tsx).

## Чек-лист при ревью блока

1. Есть ли `shadow-*` / `drop-shadow-*`? → убрать.
2. Есть ли `hover:scale-*` / `hover:translate-*` / `hover:shadow-*`? → убрать, оставить только прозрачность.
3. У элемента есть `bg-*` и `border-*`? → убрать `border-*`.
4. Кнопка primary с градиентом или тенью? → плоский фон.
5. Кнопка secondary залитая? → сделать контурной.
6. У кнопки не `rounded-full`? → исправить.
7. У секции/блока/карточки/медиа острые углы? → добавить скругление (`rounded-2xl` / `rounded-3xl`, единый в рамках экрана).
8. На `.font-heading` висит `font-bold`/`font-extrabold`/`font-semibold`? → убрать (вес 800 — инвариант).
9. Card-блок (`surface: card`) содержит вложенную карточку с `bg-slate-*`? → сменить на `bg-white` чтобы не сливалась с родителем.
10. Контур толщиной 1px (`border` без числа)? → заменить на `border-2`.
